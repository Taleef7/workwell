# Support

## How to get help

- For bugs: open a GitHub issue with reproduction details.
- For feature requests: open a GitHub issue and describe the use case, expected behavior, and acceptance criteria.
- For security concerns: follow [SECURITY.md](SECURITY.md).

## What to include

- environment (local/prod, OS, browser, Java/Node versions)
- exact endpoint/page and steps to reproduce
- expected behavior vs actual behavior
- relevant logs, screenshots, and timestamps

## Response expectations

This is an actively maintained project, but response time varies with roadmap and operational load. Well-scoped reports with clear reproduction steps are prioritized.
